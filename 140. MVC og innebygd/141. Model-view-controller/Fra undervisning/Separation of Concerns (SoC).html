Separation of Concerns (SoC)

Separation of Concerns (SoC) handler om å dele opp koden din i forskjellige deler, der hver del har sitt eget spesifikke ansvar. Dette gjør koden din mer organisert, lettere å forstå, og enklere å vedlikeholde. Når du bruker SoC i JavaScript, betyr det at du prøver å holde forskjellige typer funksjoner eller logikk adskilt fra hverandre.

Her er noen eksempler på hvordan du kan bruke Separation of Concerns i JavaScript:

1. HTML, CSS, og JavaScript adskilt:

HTML skal bare inneholde struktur og innhold.
CSS skal ta seg av hvordan ting ser ut (stiler).
JavaScript skal håndtere hvordan ting fungerer (logikk og interaktivitet).
For eksempel, du bør ikke blande mye JavaScript i HTML (som onclick attributter). I stedet, bruk JavaScript for å legge til funksjonalitet etter at HTML-en er lastet.

2. Adskillelse av funksjoner i JavaScript:

  • Datahåndtering: En del av koden (modell) kan være ansvarlig for å håndtere data, som å hente, lagre, eller oppdatere data.
  • Brukergrensesnitt: En annen del (view) kan være ansvarlig for å oppdatere det brukeren ser på skjermen, som å vise dataene i en tabell eller liste.
  • Brukerinteraksjon: En tredje del (controller) kan håndtere hendelser som skjer når brukeren klikker på knapper eller sender inn skjemaer, og kobler disse hendelsene til oppdateringer i data eller brukergrensesnitt.

Eksempel:

        
// Model - håndterer data
let todoList = [];

// View - oppdaterer brukergrensesnittet
function renderTodoList() {
    let listElement = document.getElementById('todoList');
    listElement.innerHTML = '';
    todoList.forEach(function(todo) {
        let listItem = document.createElement('li');
        listItem.textContent = todo;
        listElement.appendChild(listItem);
    });
}

// Controller - håndterer brukerinteraksjon
function addTodoItem(item) {
    todoList.push(item);
    renderTodoList(); // Oppdater visningen
}

document.getElementById('addButton').addEventListener('click', function() {
    let newItem = document.getElementById('todoInput').value;
    addTodoItem(newItem);
});

I dette eksempelet er dataene (modell) adskilt fra logikken som oppdaterer skjermen (view), og brukerinteraksjonen (controller) er også adskilt. Dette er et godt eksempel på Separation of Concerns fordi hver del av koden har sitt eget ansvar, og de påvirker hverandre uten å være blandet sammen.
